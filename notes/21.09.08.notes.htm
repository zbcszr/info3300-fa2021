Shapes and Coordinates

The coordinate system starts at the top left corner, (0,0). Positive y values move down towards the bottom. Positive x values move towards the right. We can use DOM attributes or CSS to apply styles to elements.


Paths

This is the most complicated SVG element, but also one of the most useful. It contains a tiny programming language.


Transformations

Transformations are another useful (but a bit tricky) tool. How do we stretch, rotate, and shift elements, and how does the order of these operations affect the result?

Wrapping all elements in a translate() transformation moves the coordinate system. We often want to do this to add a margin around a plot while keeping our mapping from data to x,y positions simple.


First, let's mark the origin (0,0) with crosshairs.

The g tag "groups" elements so that it is easy to assign properties to sets of elements with one command.

Now let's move the origin to the center using a translate(dx,dy) transformation.

Let's create some shapes in our translated space. Note how their position on screen is adjusted by the translation, but the cx and cy of the circle stay the same. A circle at (30,40) shows onscreen at (130,140) if you apply translate(100,100).

You can also perform a translation inside of a translation!.

Let's both perform a translation and scale one of the shapes. Transforms are processed in order, starting with the last one in the string.

The order matters! What happens if we scale first and then perform a translation? It looks weird because scale() and rotate() do this *with respect to (0,0)*. So scaling *after* moving pushes things a bit further because they expand out from (0,0).

Finally, let's try rotating our coordinate system. rotate(degrees) allows for this.

Note: It can be very confusing if you get the order wrong when rotate() gets involved! It also uses (0,0) as a point of rotation, so if you move things first then they will get spun about (0,0), leading to odd results.
